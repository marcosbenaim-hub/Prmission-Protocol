Problem We Solve

AI agents are starting to read emails, access financial data, use private APIs, and act on behalf of users.

No Explicit Consent

Most systems still do not enforce clear user approval before data is used.

No Upfront Payment

There is no built-in way to require on-chain prepayment before an agent gets access.

No Accountability

Disputes, payouts, and responsibilities are often unclear after actions are completed on-chain.

Prmission adds that missing layer: explicit consent, upfront payment via escrow, and enforceable on-chain accountability.

How It Works (3 Steps)

A simple flow for AI agents and users to exchange data access and payment on-chain.

1. Agent Requests Access

An AI agent requests specific data.

2. User Approves + Agent Escrows Payment

The agent prepays in USDC on-chain into a smart contract.

3. Settlement Happens Automatically

If terms are met, payment is released to the user on-chain. If not, funds can be disputed during a challenge review.
