The Problem

AI agents can act, but they still can’t natively settle consented data transactions on-chain.

There is no default protocol that combines consent gating, escrow, dispute handling, and payout in one deterministic flow.

Without a neutral settlement layer, users are not paid directly in their own wallets when agents access permissioned data.

Agents Need Permissioned Data

Bookings, purchases, and recommendations require user context. There is no native economic rail for this exchange.

Timing Window Is Now

The Solution

Prmission is the transaction layer below the agent stack.

Prmission does not compete with ChatGPT, Apple, or Google agent platforms.

Prmission connects AI agents and human-verified users for consented data settlements on-chain.

Users approve access, agents deposit USDC in escrow, and settlement executes on-chain.

Prmission enforces consent gating, escrow, dispute handling, and auto-settlement, with a 3% transaction fee on-chain on every settlement.

Why Prmission wins:
1) Immutable monetization: 3% transaction fee is protocol-enforced on every on-chain settlement.
2) Neutral position: we integrate under agent platforms instead of competing with them.
3) Fast compounding: as agent transactions scale, settlement events scale.

Protocol Mechanics

Five steps. Verifiable on-chain.

1grantPermission()

User defines scope, purpose, fee, duration.

2depositEscrow()

Agent locks USDC before access.

3reportOutcome()

Outcome reported after task completion.

4disputeSettlement()

24-hour dispute window.

5settle()

Contract auto-settles on-chain, pays users to their Prmission wallets, and takes Prmission’s 3% fee.

Example: $500 outcome value (user fee 2%)
User receives (2% x $500)$10.00
Protocol receives (3% x $500)$15.00
Agent receives remainder$475.00
